🗂️ Notion MCP Server

Guia de instalação — só para este projeto

📖 O que é isso?

É uma "ponte" que permite que o Claude Code converse diretamente com o seu Notion: ler páginas, criar páginas, editar conteúdo, consultar bancos de dados, tudo isso a partir do chat, sem você precisar abrir o navegador.

MCP (Model Context Protocol) é o "protocolo" — um padrão que permite ao Claude Code se conectar a ferramentas externas, como o Notion, o GitHub, etc. Pense nisso como um "plugue universal" que a Anthropic e outras empresas adotaram.

⚠️ Aviso do próprio criador (Notion)

A Notion está priorizando uma versão remota desse servidor (conexão direto pela nuvem, sem instalar nada). Esse repositório que você escolheu é a versão local (roda no seu computador) e pode ser descontinuada no futuro. Ela ainda funciona perfeitamente hoje e dá mais controle (ex: token só de leitura), mas é bom saber disso.

🏷️ Tipo de recurso e como será instalado

Servidor MCP

Caminho oficial de instalação: comando claude mcp add, que registra o servidor para rodar via npx (um jeito de executar um pacote Node.js sem precisar instalá-lo permanentemente).

🧰 Pré-requisitos (verificados no seu computador)

  • ✅ Node.js instalado — versão v24.18.0 (precisa de v18+, você tem de sobra)
  • ✅ npm instalado — versão 11.16.0
  • ✅ Claude Code instalado — versão 2.1.212

Nada precisa ser instalado. Só falta o token de acesso do Notion, que você mesmo vai gerar no site do Notion.

📝 Plano de instalação (resumo)

  1. Você cria uma "integração" no site do Notion (gera um token secreto, tipo uma senha só para o Claude usar).
  2. Você conecta essa integração às páginas do Notion que quer que o Claude possa acessar.
  3. Eu registro o servidor MCP neste projeto com claude mcp add, guardando o token com segurança.
  4. Verificamos juntos que a conexão funcionou.

🔑 Sobre o token (chave secreta)

Um token/env var é como uma senha que fica guardada numa "variável de ambiente" (uma gaveta de configuração do seu computador, não um arquivo visível) — o Claude usa essa senha para provar ao Notion que tem permissão de acessar seus dados. Ele nunca fica exposto em texto no código, nem é enviado para o GitHub.
